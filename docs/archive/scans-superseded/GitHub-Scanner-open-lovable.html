Warning Active

Unauthenticated command execution API routes

Add authentication middleware to /api/run-command and /api/run-command-v2 before any non-localhost deployment.

/api/run-command/route.ts and /api/run-command-v2/route.ts accept a JSON body {"command": "..."} and execute it via the sandbox API. No authentication check, no API key validation, no rate limiting.

Mitigating factor: Commands run inside an isolated cloud sandbox (Vercel or E2B), not on the host machine. The sandbox is ephemeral and does not access host filesystem or environment. However, the sandbox has access to whatever API keys were injected at creation time.

What this means for you: If you deploy to a publicly reachable URL, anyone can run arbitrary commands in your sandbox — potentially accessing API keys configured for that session.
